# Product Specs

One file per feature. Each spec is the contract for the feature — what the user sees, what the API does, what the DB stores. Engineers read the spec before writing the migration, service, route, validator, and test (the AGENTS.md "new feature checklist").

- [user-manual.md](../user-manual.md) — end-user guide (owners, treasurers, members)
- [user-manual.pdf](../user-manual.pdf) — printable PDF for sharing with group members
